# 写在前面

中文教程式编码笔记 (Learn to Code) 是一份技术教程文档，前身是「奶奶都能看懂的 XX」，致力于成为最易懂的编程语法教程。

:::warning
即使你是个急性子，也请不要跳过这一部分。这一部分包含关于如何食用这个网站的指南。
:::

## 为什么做这个

>「学而不思则罔，思而不学则殆」

如果学习只是阅读输入，却没有思考、加工和输出，那么质量会差很多。因此，在大部分情况下，学习完重点难点后，把它按照自己的理解**写出来**，能够大幅提升学习的效果，也就是更难遗忘。

其实这件事我在开始学习编程之初就在做了——我的[博客](https://blog.samhou.moe/)的[早期文章](https://blog.samhou.moe/Csharp%E8%AF%BB%E4%B9%A6%E7%AC%94%E8%AE%B0%EF%BC%88%E4%B8%80%EF%BC%89-%E5%AF%B9%E8%B1%A1%E4%B8%8E%E5%BC%95%E7%94%A8/)，就是学习 C# 的笔记。

但后来我发现一个问题：我的博客被大量的单纯「语法介绍」的文章填满了。我认为博客应该是一个分享「折腾过程」的地方，而不是一个描述代码写法的文档网站。因此，我一拍脑袋做了这个网站，把之前的文章独立出来，**形成完整的知识体系**。

## 这个网站适合你吗？

下面这些类型的读者可能会喜欢这个网站：

- 有一定编程基础，但是在某个语言的特定语法的学习中遇到了困难
- 已经「知其然」，但还想要「知其所以然」

如果你满足下面的任意一条，那么本站可能**不适合你**：

- 没有任何编程基础，想要从零开始学习
- 喜欢专业化，非常严谨的表述，而不是类比、大白话阐释和说明

:::tip
想要从零开始学习？直接买本相应语言的指南吧！我推荐 *Head First 系列* 的编程书籍，他们的教程生动易懂，本站的很多设计理念都是从那里来的！看到一半再回来浏览这个网站也不迟。
:::

## 内容分区介绍

如果你的屏幕够大，左侧的导航栏中应该已经显示出网站的结构了。这份教程是按照编程语言分区的，每个语言中的内容是由浅入深的，建议**按顺序阅读**。但是，如果你已经**有相当的编程基础**，那么任何内容都**没有顺序之分**。如果屏幕大小不足，可以点击左上角箭头展开。

:::danger
「旧版笔记」收集了博客上的早期作品，它们**不是教程式的**，而是单纯的笔记记录。仅作存档用途，不建议任何人阅读。
:::

选一个你感兴趣的语言开始学习吧！
